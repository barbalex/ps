# Retours et bugs

Je suis toujours heureux de recevoir des retours, suggestions, idées et signalements de bugs (malheureusement, aucun logiciel n'en est exempt).

Le meilleur moyen est de passer par GitHub, où le code est hébergé:

- Sur [GitHub issues](https://github.com/barbalex/ps/issues)
- ...veuillez d'abord vérifier si votre demande a déjà été signalée
- Si c'est le cas: participez à la discussion existante
- Sinon, vous pouvez ouvrir un nouvel issue

GitHub est idéal car il permet de garder une vue d'ensemble, de prioriser, de documenter l'évolution et de faire le lien direct avec le nouveau code. Toute personne qui participe à un issue ou s'y abonne est automatiquement notifiée par email des nouveaux commentaires — par exemple lorsqu'un bug a été corrigé.

Il faut bien sûr créer un compte GitHub pour participer à la discussion. Et celle-ci est publique. Si vous ne le souhaitez pas, vous pouvez aussi m'envoyer un [email](mailto:alex@gabriel-software.ch?subject=arten-fördern.app). Si je trouve la suggestion utile, j'ouvrirai moi-même l'issue — sans mentionner la source si vous le préférez. En général, je suis heureux de citer ceux qui ont proposé de bonnes idées.
