# Open Source

Zuletzt aktualisiert am 17. April 2026

arten-fördern.app wird von [Gabriel-Software](https://gabriel-software.ch) entwickelt.

Der Code ist [open source](https://github.com/barbalex/ps/blob/main/LICENSE). Es steht allen frei, ihn zu kopieren und die App selber zu betreiben.

Wieso mache ich das so?

1. Naturschutz dient der Natur und der Allgemeinheit. Natürlich brauche ich ein Einkommen. Ich bin aber auch Idealist: Open source Code dient der Allgemeinheit viel besser und nachhaltiger als closed source. Es ist mein Beitrag an den Naturschutz. Und ich hoffe, dass ich trotzdem davon leben kann.
2. Neben privaten Ökobüros sind öffentliche Stellen das primäre Zielpublikum für arten-fördern.app. Ich bin der Meinung, dass öffentliche Stellen möglichst ausschliesslich open source Code verwenden sollten (Idealist eben...).
3. Ich bin eine Einpersonen-Firma. Closed source Code würde mit mir sterben. Open source kann problemlos weiter betrieben und entwickelt werden.
4. Auch wenn ich aufgrund meiner Geschichte für die Entwicklung einer Naturschutz-Software sehr viel geeignetes Know-How mitbringe: Ich habe die Weisheit nicht gepachtet. In diesem open source Projekt kann jeder beitragen. Ich kann mir gut vorstellen, dass einzelne Kantone oder Ökobüros Weiterentwicklungen finanzieren, von denen dann alle profitieren können. Das funktioniert in anderen Projekten gut (z.B. QGIS).
5. Ist es nicht dumm, die Kontrolle aus der Hand zu geben? Aus egoistischer Sicht vielleicht. Ich bin aber davon überzeugt, dass Code, der nicht dauernd unterhalten und weiter entwickelt wird, tot ist. Eine App, welche sich nicht dauernd den Bedürfnissen ihrer Benutzer anpasst und sich vorausschauend weiter entwickelt, verliert ihre Daseinsberechtigung. Aus Sicht der Benutzer und Auftraggeber heisst das auch: Mich nachhaltig finanzieren lohnt sich. Habe ich euch überzeugt?
