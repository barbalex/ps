# Rückmeldungen und Fehler

Ich freue mich über Rückmeldungen, Anregungen, Ideen und Fehlermeldungen (es gibt leider keine Software ohne).

Am besten geht das auf GitHub, wo der Code gehostet wird:

- Auf [GutHub issues](https://github.com/barbalex/ps/issues)
- ...schaut ihr bitte zuerst, ob euer Anliegen schon gemeldet wurde
- Falls ja: beteiligt euch an der existierenden Diskussion
- Wenn nicht, könnt ihr einen neuen "Issue" eröffnen

GitHub eignet sich, weil man die Übersicht behält, priorisieren und den Verlauf dokumentieren kann und am Ende einen direkten Bezug zum neuen Code hat. Jeder, der sich an einem Issue beteiligt oder ihn abonniert, wird automatisch per email über neue Bemerkungen informiert, z.B. wenn ein Fehler korrigiert wurde.

Man muss natürlich auf GitHub ein Konto eröffnen, um dort an der Diskussion teilnehmen zu können. Und sie ist öffentlich. Wenn ihr das nicht wollt, könnt ihr mir gerne auch ein [Email](mailto:alex@gabriel-software.ch) schreiben. Wenn ich den Input nützlich finde, werde ich anschliessend den Issue selber eröffnen. Auf Wunsch hin ohne die Quelle zu nennen - generell sage ich gerne, wer eine gute Idee eingebracht hat.
