# Rückmeldungen und Fehler

_Zuletzt aktualisiert am 8.5.2026_

---

Ich freue mich über Rückmeldungen, Anregungen, Ideen und (_zugegeben etwas weniger_) Fehlermeldungen.

Am besten geht das auf GitHub, wo der Code gehostet wird:

- Auf [GutHub issues](https://github.com/barbalex/ps/issues)
- ...schaust du bitte zuerst, ob dein Anliegen schon eingebracht wurde
- Falls ja: beteilige dich an der existierenden Diskussion
- Wenn nicht, kannst du einen neuen "Issue" eröffnen

GitHub eignet sich, weil man die Übersicht behält, priorisieren und den Verlauf dokumentieren kann und am Ende einen direkten Bezug zum neuen Code hat. Jeder, der sich an einem Issue beteiligt oder ihn abonniert, wird automatisch per E-Mail über neue Bemerkungen informiert, z.B. wenn ein Fehler korrigiert wurde.

Du musst ein Konto eröffnen (gratis), um auf GitHub an der Diskussion teilnehmen zu können. Und sie ist öffentlich. Wenn du das nicht willst oder wenn es dir einfach zu kompliziert ist, kannst du mir gerne stattdessen eine [E-Mail](mailto:alex@gabriel-software.ch?subject=arten-fördern.app) schreiben.

Wenn ich den Input nützlich finde, werde ich anschliessend den Issue selber eröffnen. Auf Wunsch hin ohne die Quelle zu nennen. Generell sage ich aber gerne, wer eine gute Idee eingebracht hat.
