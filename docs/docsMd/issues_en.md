# Feedback and Bugs

*Last updated: 8 May 2026*

---

I welcome feedback, suggestions, ideas, and (_admittedly somewhat less_) bug reports.

The best way to do this is on GitHub, where the code is hosted:

- On [GitHub issues](https://github.com/barbalex/ps/issues)
- ...please check first whether your concern has already been reported
- If so: join the existing discussion
- If not, you can open a new issue

GitHub is ideal because it keeps things organised, allows prioritisation and documentation of progress, and ultimately links directly to the new code. Everyone who participates in or subscribes to an issue is automatically notified by email of new comments — for example when a bug has been fixed.

You need to create an account (free) to participate in the discussion on GitHub. And it is public. If you prefer not to, or if you find it simply too complicated, you are welcome to send me an [email](mailto:alex@gabriel-software.ch?subject=arten-fördern.app).

If I find the input useful, I will then open the issue myself. At your request, without mentioning the source. In general, I am happy to credit whoever contributed a good idea.
