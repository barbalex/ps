Feedback and Bugs

I welcome feedback, suggestions, ideas, and bug reports (unfortunately no software is without them).

The best way to do this is on GitHub, where the code is hosted:

  • On GitHub issues
  • ...please check first whether your concern has already been reported
  • If so: join the existing discussion
  • If not, you can open a new issue

GitHub is ideal because it keeps things organised, allows prioritisation and documentation of progress, and ultimately links directly to the new code. Everyone who participates in or subscribes to an issue is automatically notified by email of new comments — for example when a bug has been fixed.

You do of course need a GitHub account to participate in the discussion there. And it is public. If you prefer not to, you are welcome to send me an email. If I find the input useful, I will then open the issue myself — without mentioning the source if you wish. In general, I am happy to credit whoever contributed a good idea.
