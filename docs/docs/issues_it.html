Feedback e bug

Ultimo aggiornamento: 17 aprile 2026

Sono sempre felice di ricevere feedback, suggerimenti, idee e segnalazioni di bug (purtroppo nessun software ne è privo).

Il modo migliore è tramite GitHub, dove è ospitato il codice:

  • Su GitHub issues
  • ...verificate prima se la vostra segnalazione è già stata aperta
  • In caso affermativo: partecipate alla discussione esistente
  • In caso contrario, potete aprire un nuovo issue

GitHub è ideale perché permette di mantenere una visione d'insieme, di stabilire priorità, di documentare l'evoluzione e di avere alla fine un collegamento diretto con il nuovo codice. Chiunque partecipi a un issue o vi si iscriva riceve automaticamente una notifica via email per ogni nuovo commento — ad esempio quando un bug è stato corretto.

Occorre ovviamente creare un account GitHub per partecipare alla discussione. E la discussione è pubblica. Se preferite evitarlo, potete anche inviarmi una email. Se trovo il contributo utile, aprirò io stesso l'issue — senza citare la fonte se lo desiderate. In generale, sono lieto di menzionare chi ha proposto una buona idea.
