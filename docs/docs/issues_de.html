Rückmeldungen und Fehler

Zuletzt aktualisiert am 17. April 2026

Ich freue mich über Rückmeldungen, Anregungen, Ideen und (zugegeben etwas weniger) Fehlermeldungen.

Am besten geht das auf GitHub, wo der Code gehostet wird:

  • Auf GutHub issues
  • ...schaut ihr bitte zuerst, ob euer Anliegen schon gemeldet wurde
  • Falls ja: beteiligt euch an der existierenden Diskussion
  • Wenn nicht, könnt ihr einen neuen "Issue" eröffnen

GitHub eignet sich, weil man die Übersicht behält, priorisieren und den Verlauf dokumentieren kann und am Ende einen direkten Bezug zum neuen Code hat. Jeder, der sich an einem Issue beteiligt oder ihn abonniert, wird automatisch per E-Mail über neue Bemerkungen informiert, z.B. wenn ein Fehler korrigiert wurde.

Man muss natürlich auf GitHub ein Konto eröffnen, um dort an der Diskussion teilnehmen zu können. Und sie ist öffentlich. Wenn ihr das nicht wollt oder wenn es euch einfach zu kompliziert ist, könnt ihr mir gerne auch eine E-Mail schreiben.

Wenn ich den Input nützlich finde, werde ich anschliessend den Issue selber eröffnen. Auf Wunsch hin ohne die Quelle zu nennen - generell sage ich gerne, wer die gute Idee eingebracht hat.
