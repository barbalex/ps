Retours et bugs

Dernière mise à jour: 17 avril 2026

Je suis toujours heureux de recevoir des retours, suggestions, idées et (il faut l'avouer, un peu moins) des signalements de bugs.

Le meilleur moyen est de passer par GitHub, où le code est hébergé:

  • Sur GitHub issues
  • ...veuillez d'abord vérifier si votre demande a déjà été signalée
  • Si c'est le cas: participez à la discussion existante
  • Sinon, vous pouvez ouvrir un nouvel issue

GitHub est idéal car il permet de garder une vue d'ensemble, de prioriser, de documenter l'évolution et de faire le lien direct avec le nouveau code. Toute personne qui participe à un issue ou s'y abonne est automatiquement notifiée par email des nouveaux commentaires — par exemple lorsqu'un bug a été corrigé.

Il faut bien sûr créer un compte GitHub pour participer à la discussion. Et celle-ci est publique. Si vous ne le souhaitez pas, ou si vous trouvez cela simplement trop compliqué, vous pouvez aussi m'envoyer un email. Si je trouve la suggestion utile, j'ouvrirai moi-même l'issue. À votre demande, sans mentionner la source. En général, je suis heureux de citer ceux qui ont proposé de bonnes idées.
