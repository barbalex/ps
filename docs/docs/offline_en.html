Working Offline

Last updated: 17 April 2026

Why?

Nature is outdoors. You won't always have mobile reception. Not every mobile device has a SIM card.

Connection Status

arten-fördern.app shows you in the top right corner with the cloud icon whether you are online or offline.

arten-fördern.app synchronises all your data from the server into its own local database. This means you can simply keep working offline.

Operations Queue Up

When you work, arten-fördern.app packages your new data into small "parcels" called operations.

When online, these operations are immediately sent to the server and become available to all other team members right away.

When offline, the operations wait patiently — until you are online again. Then they are sent to the server immediately.

You can see how many operations are currently queued by the small number in the top right of the cloud icon. This icon is only visible when you are working with data (so not now, in the documentation). And operations should only be queued offline, so you should never see these numbers while online.

Once operations are queued, they appear in the navigation tree at the bottom left (you also only see the navigation tree when you are working with data). From there you could delete individual or all operations if needed — but you probably don't want to, as they would be lost permanently.

Conflicts

If two team members work offline on the same record at the same time, multiple versions may have been recorded. When these versions differ, we call that a conflict. The version that was last synchronised to the server "wins" — meaning it will be displayed going forward without any correction from you.

If you notice conflicts, you can resolve them easily: open the record's history using the history menu in the form. There you'll see all versions in the order they arrived at the server, how they differ from the current version, and you can correct the data directly.

Histories are stored only on the server. That is why you can see them only while online.
